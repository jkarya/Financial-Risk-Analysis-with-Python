# Financial-Risk-Analysis-with-Python


## Project Overview

This project analyzes Barclays’ transactional banking data to uncover customer behavior, transaction trends, and financial risks using Python. The analysis focuses on identifying dormant accounts, overdrafts, anomalies, customer segments, and transaction patterns to support data-driven banking decisions.

---

## Business Objective

* Analyze customer transaction behavior
* Track credit and debit trends
* Identify risky and dormant accounts
* Detect overdrafts and anomalies
* Segment customers based on activity and balance



## Key Analysis Performed

* Data Cleaning & Preprocessing
* Transaction Trend Analysis
* Dormant Account Detection
* Customer Segmentation
* Financial Risk Analysis
* Anomaly Detection
* Hypothesis Testing
* Data Visualization

---

## Key Insights

* Identified **128 dormant accounts**
* Detected **14 overdraft accounts**
* Found **44 high-volatility accounts**
* Detected **4 anomalous transactions**
* March and October showed peak transaction activity
* Mutual fund transactions had the highest engagement
